CSC 256 - Software Quality Assurance

Instructor: Cindy Halliday

Course Overview

This course covers the principles, concepts, and processes of software testing. Topics include testing technologies, static techniques, test design techniques, and test management.

Upon completion, students should be able to design and implement software testing plans and procedures throughout the software lifecycle.

Student Learning Outcomes (SLOs)

  • Understand the importance of software quality assurance and testing in the software development life cycle.
  • Apply SQA principles throughout the software lifecycle.
  • Understand and utilize different testing methodologies, including unit, integration, system, and acceptance testing.
  • Evaluate the benefits of automated testing and its role in increasing efficiency and effectiveness.
  • Develop skills in popular testing tools for both manual and automated testing.
  • Analyze API testing, its importance, and effective implementation methods.
  • Explore Continuous Integration/Continuous Delivery (CI/CD) and its role in maintaining software quality.
  • Write and manage test cases and test plans effectively.

Lesson Plans

Lesson 1: Intro and Preparation

Topics: Programming Paradigms, DevOps, Software Design Principles, Markdown, Python Preparation.

Lesson 2: SQA and Testing

Topics: Software Testing, QA vs QC, SQA Activities, STLC Stages, Git and Version Control.

Lesson 3: Testing Types and Approaches

Topics: Unit Testing, White/Black/Grey Box Testing, Integration Testing, System and Acceptance Testing.

Lesson 4: Test Plans, Test Cases, and Bug Cycle

Topics: Software Design Reviews, Test Plans, Test Cases, Requirements Traceability.

Lesson 5: Manual and Automated Testing P1

Topics: Manual Testing, Automation Frameworks, APIs and Tools, Test Data Management.

Group Project Overview

Students will work together to design and develop a web application using Python and web technologies. Parallel testing is implemented throughout to ensure continuous quality assurance.

  • Collaborative Development: Teams design and develop a functional web application.
  • Testing and Quality Assurance: Implement various tests to ensure reliability and functionality.
  • Test Coverage: Achieve and document a coverage requirement of 70-80%.
  • Documentation and Demonstration: Create documentation and Educational Student Labs.
  • Use automated testing tools for Unit, UI, API, and Acceptance Testing.
